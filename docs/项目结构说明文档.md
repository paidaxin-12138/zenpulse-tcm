# 中医AI诊断系统 - 项目结构说明文档

## 1. 项目概述
本项目是一个结合计算机视觉、生理参数采集和大语言模型的多模态智能中医诊断系统。系统通过望诊（面部、舌头、眼睛）和生理参数（心率、血压、体温）的综合分析，实现中医智能诊断和调理建议。系统整合了中医经典文献、现代医学研究和真实临床案例，提供中医知识查询、案例分析和智能诊断功能。采用模块化设计，支持知识库的独立扩展和向量化存储，便于后续功能扩展和维护。

## 3. 核心模块详细说明

### 3.1 中医知识库模块 (tcm_knowledge/)
中医知识库的核心存储目录，包含中医理论、临床案例、诊断方法等权威资料。

#### 3.1.1 知识库结构
- **基础理论**：整合《黄帝内经》、《伤寒论》等经典中医著作，提供理论基础
- **临床案例**：标准化JSON格式存储的真实临床案例，包含患者信息、症状、诊断、治疗方案等
- **体质学**：中医体质分类和调理方法
- **诊断学**：中医四诊（望、闻、问、切）的理论和实践
- **中药学**：中药性味归经、功效主治等信息
- **治疗学**：中医治疗原则和方法

#### 3.1.2 索引系统
- **多维度索引**：构建症状、证型、年龄、性别、诊断等多维度索引
- **智能检索**：支持关键词匹配和语义搜索
- **案例关联**：建立案例之间的关联关系，支持相似案例推荐

### 3.2 AI核心模块 (tcm_ai/)
系统的AI核心模块，实现多模态数据融合和智能诊断功能。

#### 3.2.1 视觉检测模块 (vision/)
- **人脸检测**：基于OpenCV Haarcascade和YOLOv8的双引擎面部检测
- **舌头检测**：基于色彩阈值分割和形态学操作的舌头区域检测
- **眼睛检测**：眼球血丝分析和眼色评估
- **特征提取**：提取面部肤色、舌色、舌苔、眼球血丝等特征

#### 3.2.2 生理数据处理模块 (stm/)
- **数据采集**：通过串口通信采集生理参数设备数据
- **数据解析**：解析心率、血压、体温等生理参数
- **状态评估**：评估生理参数状态（正常、异常）
- **模拟数据**：无设备时生成符合生理规律的模拟数据

#### 3.2.3 智能诊断模块 (ai/)
- **TCMAgent**：中医智能诊断代理，实现多模态融合诊断
- **知识管理**：知识库加载、索引构建和检索
- **多模态融合**：基于中医理论的权重分配和数据融合
- **LLM推理**：使用本地LLM进行中医诊断推理
- **规则引擎**：基于规则的备选诊断方案

#### 3.2.4 系统入口 (main.py)
- **系统初始化**：初始化各模块组件
- **摄像头管理**：检测和管理摄像头设备
- **用户交互**：处理用户输入和显示诊断结果
- **诊断流程**：协调各模块完成完整诊断流程

### 3.3 模型和存储模块

#### 3.3.1 模型文件 (models/)
- **BGE-small-zh-v1.5**：中文文本嵌入模型，用于知识向量化
- **Haarcascades**：OpenCV人脸和眼睛检测模型
- **YOLOv8n**：轻量级目标检测模型，作为备选检测方案

#### 3.3.2 向量存储 (vector_store/)
- **Chroma向量数据库**：存储知识库的向量表示
- **语义索引**：支持高效的语义相似度搜索
- **持久化存储**：向量索引持久化，支持系统重启后快速加载

### 3.4 前端和接口模块

#### 3.4.1 前端界面 (frontend/)
- **Web界面**：基于HTML/CSS/JavaScript的用户界面
- **交互功能**：提供知识库查询、案例分析等用户交互

#### 3.4.2 Web服务器 (web_server.py)
- **FastAPI框架**：构建高性能RESTful API
- **接口设计**：提供知识库查询、案例分析、诊断推理等接口
- **数据交换**：支持JSON格式的数据交换

## 4. 核心功能实现

### 4.1 多模态数据融合
- **数据预处理**：统一各模态数据格式，评估数据质量
- **权重分配**：基于中医理论的权重分配（舌诊0.35、面诊0.25等）
- **特征融合**：加权融合多模态特征，生成诊断依据
- **置信度计算**：计算诊断结果的置信度

### 4.2 知识检索增强
- **向量检索**：基于BGE模型的语义检索
- **关键词检索**：传统关键词匹配作为备选
- **知识排序**：基于相关性和权威性排序检索结果
- **上下文构建**：构建包含患者数据和相关知识的推理上下文

### 4.3 智能诊断推理
- **LLM推理**：使用本地LLM（如deepseek-r1:1.5b）进行诊断推理
- **规则引擎**：基于中医诊断规则的备选方案
- **诊断生成**：生成中医证型诊断和调理建议
- **方案推荐**：推荐中药方剂和生活调理方案

### 4.4 实时视觉分析
- **视频处理**：实时处理摄像头视频流
- **目标检测**：检测面部、舌头、眼睛等目标
- **特征分析**：实时分析视觉特征
- **结果显示**：实时显示检测结果和诊断信息

## 6. 系统架构设计

### 6.1 分层架构
- **数据源层**：摄像头、生理参数设备、知识库文件
- **数据处理层**：视觉检测、生理数据处理、知识库管理、数据融合
- **推理层**：LLM推理引擎、规则引擎
- **应用层**：诊断结果展示、用户交互界面

### 6.2 数据流
1. **数据采集**：摄像头采集视觉数据，设备采集生理数据
2. **数据预处理**：检测、分割、特征提取、数据解析
3. **多模态融合**：加权融合多模态特征
4. **知识检索**：基于融合特征检索相关中医知识
5. **诊断推理**：LLM推理生成诊断结果
6. **结果输出**：显示诊断报告和调理建议

### 6.3 扩展性设计
- **模块化结构**：各功能模块独立设计，便于扩展
- **插件架构**：支持添加新的检测算法和诊断模型
- **知识库扩展**：支持按类别添加新的知识文件
- **API接口**：基于FastAPI的RESTful API，便于与其他系统集成
- **容器化部署**：支持Docker容器化部署

## 8. 部署和维护

### 8.1 部署要求
- **硬件**：4核CPU、8GB内存、10GB存储空间
- **软件**：Windows 10/11或Ubuntu 20.04+、Python 3.10+
- **依赖**：详见requirements.txt

### 8.2 安装步骤
1. **环境准备**：创建并激活虚拟环境
2. **依赖安装**：pip install -r requirements.txt
3. **模型下载**：确保models目录包含必要的模型文件
4. **知识库准备**：确保tcm_knowledge目录包含知识库文件
5. **系统启动**：运行web_server.py启动Web服务器

### 8.3 维护说明
- **知识库更新**：添加新的知识文件后运行reset_vector_store.py重新构建向量存储
- **模型更新**：将新的模型文件添加到models目录，更新配置
- **日志管理**：系统运行日志通过FastAPI日志系统查看
- **性能监控**：监控CPU、内存使用和推理时间
- **故障排查**：使用测试脚本验证各模块功能
