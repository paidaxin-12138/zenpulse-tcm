# 真机验收清单（§11.2）

> 答辩前在 **真实 ESP32 + 双 MAX30102 + 微信小程序** 环境逐项勾选。

## 环境准备

- [ ] 后端：`python web_server.py`，局域网 IP 与 `wechat-miniprogram/config/env.js` 中 `customBaseUrl` 一致
- [ ] 固件：已烧录最新 `firmware/esp32_pulse_wrist`，BLE 已开启（`ENABLE_BLE=1`）
- [ ] 小程序：开发者工具或真机预览，已关闭「不校验合法域名」（开发阶段）
- [ ] 腕带佩戴：双通道 IR 信号在 5–26 万范围内（串口/OLED 可见）

## BLE 采集 → 分析

- [ ] 小程序「指标」页连接设备，采集 ≥20s，CH1/CH2 均有数据
- [ ] `/api/vitals/analyze` 返回 `success: true`，HR/SpO₂ 在合理范围
- [ ] 双通道长度不一致时出现告警，但不阻断成功路径（若合并后仍 ≥500 点）

## 诊断链路

- [ ] 采集完成后进入上传/诊断，`vitals_assessment` 随请求提交（非重复上传原始波形）
- [ ] `/api/diagnose/json` 返回完整结果卡，脉象与体征一致
- [ ] 断网重连、BLE 断开后重连可恢复采集

## 固件韧性

- [ ] WiFi 未配置或连接失败时：**不重启**，BLE/PPG 采样继续
- [ ] WiFi 恢复后 TCP 调试端口可用（若启用）

## 安全（生产配置抽查）

- [ ] `TCM_ENV=production` 时 `allow_public_vitals=false`
- [ ] `admin_api_key` 长度 ≥32，`dev_mode=false`
