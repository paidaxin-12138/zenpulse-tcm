# 脉象硬件腕带方案（桡动脉 · L1/L2）

> **完整开发流程**：[硬件开发指南.md](./硬件开发指南.md)  
> 配套固件：`firmware/esp32_pulse_wrist/`  
> 算法与能力分级：[脉象算法开发文档.md](./脉象算法开发文档.md)

本文描述 **腕部寸口桡动脉** PPG 采集，不使用指夹式。指夹虽易买，但与「腕脉 / 切诊」叙事不一致，且难以固定取脉位置。

---

## 1. 方案概览

```
┌─────────────────────────────────────────┐
│  魔术贴腕带（可调周长 14–22 cm）          │
│  ┌─────────────────────────────────┐   │
│  │ 3D 打印托板（刚性）               │   │
│  │  [MAX30102 #1]  [MAX30102 #2]    │   │  ← 窗口贴皮肤，间距 8–12 mm
│  │       ↑ 硅胶遮光圈                │   │
│  └─────────────────────────────────┘   │
│  ESP32 模块（可放托板背面或小盒）        │
└─────────────────────────────────────────┘
         │ I2C + 3.3V
         ▼
    笔记本 / 手机热点 / 路由器
         │ WiFi TCP :8080
         ▼
    ZenPulse 后端 (STMDataProcessor)
```

| 等级 | 本方案包含 | 后续扩展 |
|------|------------|----------|
| **L1** | 双 MAX30102 + 腕带 + ESP32 | — |
| **L2** | + MPU6050 同 I2C 总线 | 固件协议加 AX/AY/AZ |
| **L3** | 另做加压腕托（见算法文档 §3.5） | 不与 L1 腕带共用结构 |

## 3. 接线

### 3.1 I2C 总线（双 MAX30102）

| ESP32-S3 | MAX30102 #1 | MAX30102 #2 | OLED SSD1306 | MPU6050 (L2) |
|----------|-------------|-------------|--------------|--------------|
| 3.3V | VIN | VIN | VCC | VCC |
| GND | GND | GND | GND | GND |
| **GPIO8 (SDA)** | SDA | SDA | SDA | SDA |
| **GPIO9 (SCL)** | SCL | SCL | SCL | SCL |

- OLED 与 PPG **共用 I2C 总线**（地址 0x3C，与 0x57/0xAE 不冲突）
- 屏体建议装在 **腕带外侧主控仓**，传感器仍在内侧贴皮肤

经典 ESP32 见 [BOARDS.md](../firmware/esp32_pulse_wrist/BOARDS.md)（GPIO21/22）。

- 线长 **< 15 cm**，远离 USB 高频开关噪声。
- 模块间 **并联** 同一 I2C；#1 地址 `0x57`，#2 **`0xAE`**（改 SDO 焊盘，见模块说明书）。

### 3.2 供电

- 开发阶段：USB 5V 供电即可。
- 佩戴原型：5V 充电宝 + USB 短线；ESP32 3.3V LDO 供 MAX30102。
- MAX30102 LED 电流较大，**勿**用 ESP32 3.3V 引脚直接带两个模块的 VIN（应走开发板 3.3V 电源轨，且注意发热）。

## 4. 机械结构

### 4.1 取脉位置（左右手对称设计）

腕带佩戴于 **桡骨茎突内侧**，传感器窗口对准 **桡动脉搏动区**：

- **#1 主传感器**：略偏 **关部**（桡骨茎突内侧凹陷处），为临床常用取脉点。
- **#2 冗余传感器**：沿动脉走向偏近心 **5–10 mm**，用于双路一致性质控与平均降噪。

用户说明（可印在小卡片上）：

1. 坐位，前臂放桌上，与心脏同高  
2. 腕带松紧以 **能塞进一根手指** 为宜，不可压死  
3. 测量 30 s **保持静止**

### 4.2 3D 打印托板要点

| 参数 | 建议值 |
|------|--------|
| 材料 | PLA / PETG |
| 尺寸 | 约 50 × 30 × 8 mm（按模块实际改） |
| 传感器窗口 | LED 正对皮肤，周围 **2 mm 遮光裙边** |
| 曲率 | 底面微弧 R≈40 mm，贴合腕背 |
| 固定 | 腕带穿过两端槽位；魔术贴不与传感器窗口重叠 |

可将两个 MAX30102 并排在托板底面；ESP32 用尼龙柱固定在托板顶面或延长线接到小盒。

### 4.3 遮光

环境光进入 RED/IR 会导致波形失真。每个传感器加 **黑色硅胶圈** 贴皮肤一圈密封。

## 5. 固件与协议

### 5.1 烧录

见 [firmware/esp32_pulse_wrist/README.md](../firmware/esp32_pulse_wrist/README.md)。

### 5.2 数据格式（v0，已实现）

```
CH1:123456,CH2:123012\n
```

- 频率：100 Hz（`config.h` 中 `SAMPLE_HZ`）
- 数值：MAX30102 **IR 原始计数**（后端做 AC/DC 分离与滤波）
- 传输：WiFi TCP Server，端口 **8080**；Python 端主动 `connect()` 后接收

### 5.3 计划扩展（L2）

```
CH1:123456,CH2:123012,AX:12,AY:-3,AZ:980\n
```

ACC 单位 m/s² 或 raw，需与 PPG **时间戳对齐**（固件升级时加 `ts_ms` 字段）。

## 6. 后端联调

```bash
# .env 或 export
TCM_STM_ENABLED=true
TCM_STM_HOST=192.168.x.x    # ESP32 串口打印的 IP
TCM_STM_PORT=8080
```

Python 类：`tcm_ai/adapters/stm/processor.py` → `STMDataProcessor`

- 缓冲 ≥100 点后每 5 s 处理一次  
- Phase 1 算法接入后，建议改为 **整段 30 s 上传** 至 `POST /api/pulse/analyze`

## 7. 调试检查表

| 现象 | 可能原因 | 处理 |
|------|----------|------|
| 串口 I2C 失败 | 接线/地址冲突 | 单颗逐个点亮；Scanner 扫 0x57/0xAE |
| IR 恒为 0 或极小 | 未贴皮肤、LED 未开 | 手指覆盖测试；调 `LED_BRIGHTNESS` |
| IR 饱和不变 | 过紧/LED 过大 | 松腕带、降 LED |
| 波形噪声大 | 环境光、50 Hz | 遮光；后端陷波 |
| TCP 无数据 | 防火墙/未 connect | `nc IP 8080`；确保有客户端连接后才开始推流 |
| CH1≈CH2 完全相同 | 只读到一个地址 | 改第二颗模块地址 |

## 8. 与指夹方案对比（为何选腕带）

| 项目 | 腕带（本方案） | 指夹 |
|------|----------------|------|
| 与中医「寸口脉」 | ✅ 一致 | ❌ 为指尖血氧设计 |
| 固定取脉位置 | 托板 + 腕带可重复 | 夹力因人而异 |
| 双传感器布局 | 沿动脉走向 | 难 |
| 后续 L3 加压 | 可升级腕托 | 指夹空间不足 |
| 初次搭建 | 需 3D 打印/腕带 | 即插即用 |
