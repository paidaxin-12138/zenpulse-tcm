# 中医AI诊断系统微信小程序设计方案

> **v1.0 更新（2026-Q2）**  
> 主路径改为 **MAX30102 蓝牙生理参数** + 望诊 + 诊断融合；用户侧输出 **`vitals_assessment`**，不再展示中医脉名（迟/数/滑/涩）。  
> 技术细节见 **[脉象算法开发文档.md](./脉象算法开发文档.md)**。

## 1. 系统架构

### 1.1 整体架构
```
┌─────────────────┐       ┌─────────────────┐       ┌─────────────────┐
│ 微信小程序前端  │ <────> │  FastAPI后端    │ <────> │  知识库 + RAG   │
│ (BLE + 望诊)    │       │  + VitalsService│       │  + LLM 辨证     │
└─────────────────┘       └─────────────────┘       └─────────────────┘
         │                          │
         └──── ESP32 ZenPulse ──────┘  （BLE GATT，可选）
```

### 1.2 技术栈
- **前端**：微信小程序原生框架（WXML、WXSS、JavaScript）
- **后端**：FastAPI（Python 3.10+）
- **体征**：MAX30102 → `utils/bleVitals.js` → `POST /api/vitals/analyze`
- **数据传输**：JSON + multipart 图片上传

## 2. 页面结构

### 2.1 页面列表

| 页面名称 | 路径 | 功能描述 |
|---------|------|----------|


## 3. 功能模块设计

### 3.1 首页模块
- 系统介绍和功能说明
- 开始诊断按钮
- 历史记录入口

### 3.2 健康指标模块
- 心率 / 脉搏 / 血压 / 年龄 / 性别
- **蓝牙采集**：连接 ZenPulse 腕带 ~10s，`analyzeVitals()` 回填 HR/SpO2
- 隐私授权：`utils/privacy.js` + `agreePrivacyAuthorization` 按钮
- 下一步 → 望诊上传

### 3.3 图片上传模块
- 舌苔 / 面部 / 眼部照片（`chooseMedia`）
- 调用 `wxDiagnose()` → `POST /api/diagnose/json`

### 3.4 诊断结果模块
- 证候、分析、建议、方剂
- **`vitals_assessment`**：心率、血氧、综合状态
- 报告导出 PNG（`reportExport.js`，含生理参数）
- **不展示** `pulse_characteristics` 脉象类型

### 3.5 个人中心模块
- 微信登录（`POST /api/wx/login`）
- 历史记录（`/api/diagnosis/history` 或本地存储）

## 4. API接口设计

### 4.1 诊断与体征

| API路径 | 方法 | 功能描述 |
|---------|------|----------|
| `/api/diagnose/json` | POST | JSON 诊断（体征 + 可选 base64 图片 + 波形） |
| `/api/diagnose` | POST | multipart 诊断（图片 + 表单体征） |
| `/api/vitals/analyze` | POST | MAX30102 波形 → HR/SpO2 规则评估 |
| `/api/health-metrics` | GET | 随机演示用健康指标 |

### 4.2 微信与用户

| API路径 | 方法 | 功能描述 |
|---------|------|----------|
| `/api/wx/login` | POST | 微信 code 换 Token |
| `/api/wx/refresh` | POST | Token 刷新 |
| `/api/diagnosis/history` | GET/POST | 诊断历史（需 Bearer Token） |

### 4.3 知识库

| API路径 | 方法 | 功能描述 |
|---------|------|----------|
| `/api/knowledge/search` | POST | 向量检索 + rerank |

## 6. 毕设部署说明

- **无需正式上架**：使用开发者工具 / 体验版 + 局域网后端即可答辩
- **本地后端**：`python3 web_server.py`，`config/env.js` 填电脑局域网 IP
- **开发者工具**：勾选「不校验合法域名」
- 医疗类目小程序无法过审属平台限制，毕设以 **原型演示** 为定位

## 7. 注意事项

1. **蓝牙隐私**：公众平台「用户隐私保护指引」需声明蓝牙（正式版）；毕设体验版可开发模式演示
2. **图片上传**：单文件大小限制见微信文档
3. **生产环境**：`allow_public_vitals` 建议关闭，小程序携带 Bearer Token 调用 API

## 8. 预期效果

用户可在手机上完成：**蓝牙/手动体征 → 望诊拍照 → AI 辨证结果 → 历史与报告导出**。Web 端（`frontend/`）可并行演示知识库与诊断能力。
