# 中医AI诊断系统微信小程序设计方案

## 1. 系统架构

### 1.1 整体架构
```
┌─────────────────┐       ┌─────────────────┐       ┌─────────────────┐
│ 微信小程序前端  │ <────> │  FastAPI后端    │ <────> │  中医AI模型     │
└─────────────────┘       └─────────────────┘       └─────────────────┘
```

### 1.2 技术栈
- **前端**：微信小程序原生框架（WXML、WXSS、JavaScript）
- **后端**：FastAPI（Python 3.8+）
- **AI模型**：现有的中医AI诊断模型
- **数据传输**：JSON格式
- **图片处理**：微信小程序上传API + 后端OpenCV处理

## 2. 页面结构

### 2.1 页面列表

| 页面名称 | 路径 | 功能描述 |
|---------|------|----------|
| 首页 | `/pages/index/index` | 系统介绍、主要功能入口 |
| 健康指标页 | `/pages/metrics/metrics` | 输入或获取健康指标数据 |
| 图片上传页 | `/pages/upload/upload` | 上传舌苔、面部和眼部照片 |
| 诊断结果页 | `/pages/result/result` | 显示诊断结果和脉象分析 |
| 个人中心页 | `/pages/profile/profile` | 用户信息、历史诊断记录 |

### 2.2 页面流程图

```
首页 → 健康指标页 → 图片上传页 → 诊断结果页
          ↑                    |
          └────────────────────┘
                    |
                    ↓
              个人中心页
```

## 4. API接口设计

### 4.1 现有API接口

| API路径 | 方法 | 功能描述 |
|---------|------|----------|
| `/api/health-metrics` | GET | 生成随机健康指标 |
| `/api/diagnose` | POST | 处理图片上传和诊断 |

### 4.2 微信小程序专用API

| API路径 | 方法 | 功能描述 |
|---------|------|----------|
| `/api/wx/diagnose` | POST | 处理微信小程序的诊断请求 |
| `/api/wx/history` | GET | 获取用户历史诊断记录 |
| `/api/wx/history` | POST | 保存诊断记录 |

## 7. 注意事项

1. **微信小程序域名配置**：需要在微信公众平台配置后端服务器域名
2. **图片上传限制**：微信小程序有上传文件大小限制（最大50MB）
3. **用户隐私保护**：需要遵守微信小程序用户隐私政策
4. **API调用频率**：需要合理设计API调用频率，避免触发微信小程序的限制
5. **错误处理**：需要完善前端和后端的错误处理机制

## 8. 预期效果

通过微信小程序的实现，用户可以：
1. 方便地在手机上使用中医AI诊断系统
2. 快速上传舌苔、面部和眼部照片
3. 查看详细的诊断结果和脉象分析
4. 保存和查看历史诊断记录
5. 分享诊断结果给医生或家人

同时，web页面仍然可以正常使用，实现多端访问的目标。
