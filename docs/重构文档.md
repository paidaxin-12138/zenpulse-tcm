## 1. 文档目的

本文档在以下基础上整理：

1. **首次代码审查**暴露的架构与正确性问题  
2. **管理端改进方向**：Admin UI、Admin API Key、单一流程 RAG（Embedding → Rerank → LLM）、可配置 Ollama / OpenAI 兼容 API

## 3. 重构目标架构

### 3.1 原则

1. **单一 RAG 引擎**：管理端与诊断端共用 `RAGPipeline`，不再在 `TCMAgent` 内嵌向量库逻辑  
2. **单一配置源**：`data/admin_config.json`（或演进为 `settings.yaml`）驱动 Embedding / LLM / Rerank  
3. **API 薄、Service 厚**：路由只做校验与编排  
4. **索引离线构建、在线只读**：重建仅通过 admin 或 CLI 脚本触发  
5. **结构化契约**：诊断与 RAG 返回 JSON Schema，而非纯 Markdown

### 3.3 目标数据流

#### 管理端 RAG（已实现，待统一）

```
Admin UI → /api/admin/rag/query
         → RAGPipeline.query()
         → VectorIndexService.search()
         → rerank_documents()
         → invoke_llm()
         → 结构化响应
```

#### 诊断端（目标态）

```
C 端 → /api/diagnose
     → DiagnosisService.run()
     → VisionService.analyze()
     → PulseService.analyze()
     → FusionEngine.fuse()
     → RAGPipeline.query()        # 检索知识，非重复实现
     → LLM / RuleEngine fallback
     → DiagnosisResult (JSON)
```

## 4. 关键重构项详解

### 4.1 统一 RAG 与向量索引（最高优先级）

**现状问题**

- `TCMAgent._initialize_knowledge_base()` 每次删除 `vector_store/tcm_knowledge_index` 并全量 embedding  
- `VectorIndexService` 正确实现 load-or-build，但仅管理端使用  
- 两者共用同一路径 `vector_store/tcm_knowledge_index`，存在竞态与互相覆盖  

**重构动作**

| 步骤 | 内容 | 负责模块 |
|------|------|----------|
| R-1 | 删除 `TCMAgent` 内所有 Chroma 初始化/embedding 代码 | `tcm_agent.py` |
| R-2 | `TCMAgent` 改为注入 `RAGPipeline` 或 `VectorIndexService` | `tcm_agent.py` |
| R-3 | `web_server` 启动时**不再** eager 初始化 `TCMAgent`；改用 `lifespan` + lazy | `web_server.py` |
| R-4 | 索引版本文件 `vector_store/index_meta.json`（知识库 hash、embedding 模型、chunk 参数） | `vector_index.py` |
| R-5 | 提供 `scripts/build_index.py`，CI/部署前离线构建 | `scripts/` |

**验收标准**

- 重启服务不触发 embedding  
- 管理端 rebuild 后，诊断端检索结果一致  
- 配置变更 embedding provider 后，提示需 rebuild

### 4.2 统一配置系统

**现状**

| 配置来源 | 内容 |
|----------|------|
| `data/admin_config.json` | embedding / llm / rerank / admin key |
| `tcm_ai/config/config.py` | 几乎未使用，STM COM3 等与实现不符 |
| `TCMAgent.__init__` 硬编码 | `deepseek-r1:1.5b`、BGE 路径 |

**目标**

```python

def get_settings() -> AdminConfig:

```

- 删除或废弃 `config.py` 中的重复项  
- `TCMAgent`、`providers.py`、`VectorIndexService` 全部 `get_settings()`  
- 小程序 baseUrl 移入构建配置或环境变量

**当前职责（应拆分）**

```

├── 知识管理        → KnowledgeService（已有 tcm_knowledge_loader）
```
