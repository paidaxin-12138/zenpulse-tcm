## 3. 电路与接线

### 3.1 原理框图

```
                    ┌──────────────────┐
   USB ────────────►│  ESP32-S3        │
                    │  3.3V            ├─── 3.3V ──┬── MAX30102 #1 (0x57)
                    │  GPIO8  SDA      ├───────────┼── MAX30102 #2 (0xAE)
                    │  GPIO9  SCL      ├───────────┼── MPU6050 (0x68) [L2]
                    │  WiFi            │           └── GND 共地
                    └──────────────────┘
```

> 经典 ESP32-WROOM-32 改用 **GPIO21/22** 作 I2C，见 [BOARDS.md](../firmware/esp32_pulse_wrist/BOARDS.md)。

### 3.2 引脚表（ESP32-S3 DevKitC-1，默认）

| ESP32-S3 | 连接 |
|----------|------|
| 3.3V | 所有模块 VIN/VCC |
| GND | 所有模块 GND |
| **GPIO8** | SDA |
| **GPIO9** | SCL |

**禁用 GPIO**：19/20（USB）；26–32（Flash/PSRAM 占用，勿接外围）。

**禁止**：用 GPIO 直接给 MAX30102 供电（电流不足/噪声大）。

### 3.3 双 MAX30102 改 I2C 地址

1. 第一颗：**默认** `0x57`（SparkFun 库写法，部分模块文档写 0xAE 为 7bit 地址，以库为准）  
2. 第二颗：按模块丝印 **焊接 SDO → VCC** 或切断 ADDR 跳线，变为 **`0xAE`**  
3. 烧录 [`tools/i2c_scanner`](../firmware/tools/i2c_scanner/) 应看到 **两个不同地址**

常见模块地址对照（SparkFun MAX30105 库）：

| 硬件状态 | 库地址参数 |
|----------|------------|
| SDO 接 GND | `0x57` |
| SDO 接 VCC | `0xAE` |

## 4. 固件开发

### 4.1 环境搭建

**方式 A — Arduino IDE（ESP32-S3）**

1. 安装 ESP32 板支持包 ≥ 3.0  
2. 开发板：**ESP32S3 Dev Module**  
3. **USB CDC On Boot：Enabled**（否则 Serial 无输出）  
4. Library Manager：`SparkFun MAX3010x Pulse and Proximity Sensor Library`  
5. `config.h` 中 `USE_ESP32_S3 1`，I2C 为 8/9  

**方式 B — PlatformIO（推荐）**

```bash
cd firmware/esp32_pulse_wrist
cp config.example.h config.h   # 填 WiFi
pio run -e esp32-s3-devkitc-1 -t upload
pio device monitor
```

### 4.2 配置

```bash
cp firmware/esp32_pulse_wrist/config.example.h firmware/esp32_pulse_wrist/config.h
# 编辑 WiFi、可选 BACKEND_URL、ENABLE_IMU
```

| 宏 | 说明 |
|----|------|
| `WIFI_SSID` / `WIFI_PASSWORD` | 与电脑/手机同一局域网 |
| `TCP_PORT` | 默认 8080，与 Python `STMDataProcessor` 一致 |
| `SAMPLE_HZ` | 100 |
| `ENABLE_IMU` | 1 启用 MPU6050 + 协议 v2 |
| `BACKEND_URL` | 可选，如 `http://192.168.1.7:8000/api/pulse/analyze` |

### 4.3 协议

**v0/v1 流式（TCP，已实现）**

```
CH1:123456,CH2:123012\n
```

**v2（ENABLE_IMU=1）**

```
TS:12345678,CH1:123456,CH2:123012,AX:12,AY:-3,AZ:16384\n
```

- `TS`：毫秒时间戳（`millis()`）  
- `AX/AY/AZ`：MPU6050 原始加速度（后端可换算）

**TCP 命令（连接后发送一行）**

| 命令 | 行为 |
|------|------|
| `STREAM` | 默认连续推流 |
| `CAPTURE 20` | 缓冲 20 秒后再推流（便于对齐采集窗口） |
| `PING` | 回复 `PONG\n` |

### 4.4 烧录与串口验证

1. 上传 `esp32_pulse_wrist.ino`  
2. 115200 波特率打开串口，应见 `[OK] IP: 192.168.x.x`  
3. 手指覆盖传感器，IR 值应随脉搏变化（每秒一行 debug）

## 5. 台架测试（H0–H1）

### 5.1 I2C 扫描

烧录 `firmware/tools/i2c_scanner/i2c_scanner.ino`，串口应列出 `0x57` 与 `0xAE`（及可选 `0x68`）。

### 5.2 TCP 流测试

```bash
nc 192.168.1.50 8080
# 每 10ms 左右一行 CH1/CH2
```

### 5.3 Python 采集 30 秒并分析

```bash
pip install requests  # 若未装
python3 scripts/hardware/pulse_collector.py \
  --host 192.168.1.50 --port 8080 \
  --duration 30 --analyze \
  --api http://127.0.0.1:8000/api/pulse/analyze \
  --out data/pulse_capture.csv
```

输出：CSV 文件 + 终端打印 `pulse_type`、心率、有效心搏数。

### 5.4 对接现有 STM 处理器

```bash
export TCM_STM_HOST=192.168.1.50
python3 -c "
from tcm_ai.adapters.stm.processor import STMDataProcessor
p = STMDataProcessor(host='192.168.1.50')
p.connect()
import time; time.sleep(8)
print(p.read_pulse_data())
"
```

## 6. 机械装配（H2）

详见 [hardware/wrist_bracket/SPEC.md](../hardware/wrist_bracket/SPEC.md)。

**装配顺序**

1. MAX30102 用 M2 铜柱/热熔螺母固定到托板底面（窗口对准弧面）  
2. 传感器窗口贴 **遮光 O 圈**  
3. ESP32 固定在托板顶面或 15 cm 延长线到腰盒  
4. 腕带穿过托板两侧槽，**传感器区避开魔术贴毛面**  
5. 佩戴试 fit：能塞进一根手指，LED 区紧贴桡动脉皮肤

**取脉位置**：桡骨茎突 **内侧**，#1 对关部，#2 向近心 8 mm。

## 7. 质控标准（硬件验收）

| 项 | 合格标准 |
|----|----------|
| 静止 30 s IR | 波形肉眼可见周期性，非平线 |
| 双通道 | CH1 与 CH2 不完全相同，相关系数 > 0.7 |
| 心率 | 与血氧仪/听诊对比误差 ≤ ±5 bpm |
| 晃动 | IR 幅值突变；后端 `quality` 应降分或 422 |
| 遮光 | 强光下开灯 vs 遮光套，SNR 明显改善 |

## 8. 故障排查

| 现象 | 检查 |
|------|------|
| 只有一个 I2C 地址 | 改第二颗 SDO；换线；降 I2C 400k→100k |
| WiFi 连上无 TCP 数据 | 需 **客户端先 connect** 才开始推流 |
| IR 全 0 | 3.3V 供电、手指贴合、增大 LED 亮度 |
| 后端 422 有效搏不足 | 延长采集到 30 s；检查佩戴松紧与静止 |
| ESP 发热 | 降低 LED 电流；间歇采集 |

## 9. 与小程序/后端数据流

```
腕带 ESP32 ──WiFi TCP──► pulse_collector.py ──POST──► /api/pulse/analyze
                              │
                              └── CSV 存档（标定数据集）

或：

小程序演示模式 ──POST──► /api/pulse/analyze（无硬件）
诊断 JSON ──pulse_waveform──► DiagnosisService.resolve_pulse()
```

硬件就绪后，小程序 `startWristPulseCapture` 可改为请求 **局域网网关** 或 **手机热点上的采集服务**，不再使用 `generateDemoWaveform`。
